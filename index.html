II.2

SessionList — Moniteurs Java

On reste fidèle au squelette : buf est un ArrayList<Session>, le constructeur itère de 0 à nbSlots avec buf.add(i, null). La moitié du buffer (indices 0..n/2-1) est réservée aux profs, l'autre moitié aux étudiants. On utilise synchronized + wait() / notifyAll(). 
public class SessionList<Session> {

    private ArrayList<Session> buf;

    SessionList(int nbSlots) {
        int i;
        this.buf = new ArrayList<SessionList>(nbSlots);
        for (i = 0; i < nbSlots; i++) {
            buf.add(i, null);
        }
    }

    // Indices 0 .. nbSlots/2 - 1  →  zone "prof"
    // Indices nbSlots/2 .. nbSlots-1  →  zone "student"

    public synchronized Session getProfSession()
            throws InterruptedException {
        int slot = -1;
        // cherche un slot prof occupé (index < buf.size()/2)
        while (slot == -1) {
            for (int i = 0; i < buf.size() / 2; i++) {
                if (buf.get(i) != null) { slot = i; break; }
            }
            if (slot == -1) wait();
        }
        Session s = buf.get(slot);
        buf.set(slot, null);   // libère la place (remet null)
        notifyAll();
        return s;
    }

    public synchronized Session getStudentSession()
            throws InterruptedException {
        int slot = -1;
        while (slot == -1) {
            for (int i = buf.size() / 2; i < buf.size(); i++) {
                if (buf.get(i) != null) { slot = i; break; }
            }
            if (slot == -1) wait();
        }
        Session s = buf.get(slot);
        buf.set(slot, null);
        notifyAll();
        return s;
    }

    public synchronized void addSession(Session s)
            throws InterruptedException {
        int debut, fin, slot;
        if (s.type.equals("prof")) {
            debut = 0;  fin = buf.size() / 2;
        } else {
            debut = buf.size() / 2;  fin = buf.size();
        }
        slot = -1;
        while (slot == -1) {
            for (int i = debut; i < fin; i++) {
                if (buf.get(i) == null) { slot = i; break; }
            }
            if (slot == -1) wait();
        }
        buf.set(slot, s);
        notifyAll();
    }
}
II.3

SessionList — Sémaphores Java

Même structure de buf avec indices. On remplace synchronized/wait/notifyAll par des Semaphore : un sémaphore de places libres et un de places occupées, par zone. Le mutex protège les accès concurrents à buf. 
import java.util.concurrent.Semaphore;

public class SessionList<Session> {

    private ArrayList<Session> buf;
    private final Semaphore mutex        = new Semaphore(1);
    private final Semaphore videProf;    // places libres zone prof
    private final Semaphore pleinProf    = new Semaphore(0); // sessions dispo
    private final Semaphore videStudent;
    private final Semaphore pleinStudent = new Semaphore(0);

    SessionList(int nbSlots) {
        int i;
        this.buf = new ArrayList<SessionList>(nbSlots);
        for (i = 0; i < nbSlots; i++) {
            buf.add(i, null);
        }
        this.videProf    = new Semaphore(nbSlots / 2);
        this.videStudent = new Semaphore(nbSlots / 2);
    }

    public Session getProfSession()
            throws InterruptedException {
        pleinProf.acquire();       // attend une session prof dispo
        mutex.acquire();
        Session s = null;
        for (int i = 0; i < buf.size() / 2; i++) {
            if (buf.get(i) != null) {
                s = buf.get(i);
                buf.set(i, null);
                break;
            }
        }
        mutex.release();
        videProf.release();        // libère une place prof
        return s;
    }

    public Session getStudentSession()
            throws InterruptedException {
        pleinStudent.acquire();
        mutex.acquire();
        Session s = null;
        for (int i = buf.size() / 2; i < buf.size(); i++) {
            if (buf.get(i) != null) {
                s = buf.get(i);
                buf.set(i, null);
                break;
            }
        }
        mutex.release();
        videStudent.release();
        return s;
    }

    public void addSession(Session s)
            throws InterruptedException {
        int debut, fin;
        Semaphore vide, plein;
        if (s.type.equals("prof")) {
            debut = 0;  fin = buf.size() / 2;
            vide = videProf;  plein = pleinProf;
        } else {
            debut = buf.size() / 2;  fin = buf.size();
            vide = videStudent;  plein = pleinStudent;
        }
        vide.acquire();            // attend une place libre
        mutex.acquire();
        for (int i = debut; i < fin; i++) {
            if (buf.get(i) == null) {
                buf.set(i, s);
                break;
            }
        }
        mutex.release();
        plein.release();           // signale une session disponible
    }
}
II.4

SessionList — Sans exclusion mutuelle entre types

Deux sémaphores mutex séparés (mutexProf et mutexStudent) protègent chaque zone indépendamment. Un ProfSessionHandler et un StudentSessionHandler peuvent donc s'exécuter simultanément sans se bloquer mutuellement. 
import java.util.concurrent.Semaphore;

public class SessionList<Session> {

    private ArrayList<Session> buf;
    // Un mutex par zone → pas d'exclusion mutuelle entre types
    private final Semaphore mutexProf    = new Semaphore(1);
    private final Semaphore mutexStudent = new Semaphore(1);
    private final Semaphore videProf;
    private final Semaphore pleinProf    = new Semaphore(0);
    private final Semaphore videStudent;
    private final Semaphore pleinStudent = new Semaphore(0);

    SessionList(int nbSlots) {
        int i;
        this.buf = new ArrayList<SessionList>(nbSlots);
        for (i = 0; i < nbSlots; i++) {
            buf.add(i, null);
        }
        this.videProf    = new Semaphore(nbSlots / 2);
        this.videStudent = new Semaphore(nbSlots / 2);
    }

    public Session getProfSession()
            throws InterruptedException {
        pleinProf.acquire();
        mutexProf.acquire();       // verrou zone prof uniquement
        Session s = null;
        for (int i = 0; i < buf.size() / 2; i++) {
            if (buf.get(i) != null) {
                s = buf.get(i);  buf.set(i, null);  break;
            }
        }
        mutexProf.release();
        videProf.release();
        return s;
    }

    public Session getStudentSession()
            throws InterruptedException {
        pleinStudent.acquire();
        mutexStudent.acquire();    // verrou zone student uniquement
        Session s = null;
        for (int i = buf.size() / 2; i < buf.size(); i++) {
            if (buf.get(i) != null) {
                s = buf.get(i);  buf.set(i, null);  break;
            }
        }
        mutexStudent.release();
        videStudent.release();
        return s;
    }

    public void addSession(Session s)
            throws InterruptedException {
        if (s.type.equals("prof")) {
            videProf.acquire();
            mutexProf.acquire();
            for (int i = 0; i < buf.size() / 2; i++) {
                if (buf.get(i) == null) { buf.set(i, s); break; }
            }
            mutexProf.release();
            pleinProf.release();
        } else {
            videStudent.acquire();
            mutexStudent.acquire();
            for (int i = buf.size() / 2; i < buf.size(); i++) {
                if (buf.get(i) == null) { buf.set(i, s); break; }
            }
            mutexStudent.release();
            pleinStudent.release();
        }
    }
}
II.2

VideoCatalog — Priorité aux lecteurs (moniteurs Java)

On reste fidèle au squelette : metadata est un HashMap, les deux méthodes sont readMetadata et updateMetadata. On ajoute nbReaders et writing. Les lecteurs ne bloquent que si un rédacteur écrit. Les rédacteurs attendent nbReaders == 0 && !writing. 
public class VideoCatalog {

    private HashMap<String, VideoMetadata> metadata;
    private int     nbReaders = 0;
    private boolean writing   = false;

    VideoCatalog() {
        this.metadata = new HashMap<>();
    }

    // Lecture des metadonnees d'une video (peut etre concurrent avec d'autres lectures)
    public synchronized VideoMetadata readMetadata(String videoId)
            throws InterruptedException {
        // Priorité lecteurs : on bloque seulement si un rédacteur écrit
        while (writing)
            wait();
        nbReaders++;               // on entre en lecture
        notifyAll();

        // Acces a la ressource partagee
        VideoMetadata result = metadata.get(videoId);

        nbReaders--;
        if (nbReaders == 0) notifyAll(); // réveille les rédacteurs si plus de lecteurs
        return result;
    }

    // Mise a jour des metadonnees (doit etre exclusif)
    public synchronized void updateMetadata(String videoId,
                                              VideoMetadata newData)
            throws InterruptedException {
        // Attend qu'aucun lecteur ne lise et qu'aucun rédacteur n'écrive
        while (writing || nbReaders > 0)
            wait();
        writing = true;

        // Acces exclusif a la ressource partagee
        metadata.put(videoId, newData);

        writing = false;
        notifyAll();
    }
}
II.3

VideoCatalog — Priorité aux rédacteurs

Modification : ajout de nbWritersWaiting. Dans readMetadata, on bloque aussi si des rédacteurs attendent (nbWritersWaiting > 0), même si personne n'écrit encore. Ainsi, dès qu'un rédacteur annonce son intention, les nouveaux lecteurs sont mis en attente. 
public class VideoCatalog {

    private HashMap<String, VideoMetadata> metadata;
    private int     nbReaders        = 0;
    private boolean writing          = false;
    private int     nbWritersWaiting = 0; // NOUVEAU

    VideoCatalog() {
        this.metadata = new HashMap<>();
    }

    // Lecture des metadonnees d'une video (peut etre concurrent avec d'autres lectures)
    public synchronized VideoMetadata readMetadata(String videoId)
            throws InterruptedException {
        // Priorité rédacteurs : bloqué si un rédacteur écrit OU attend
        while (writing || nbWritersWaiting > 0)
            wait();
        nbReaders++;
        notifyAll();

        // Acces a la ressource partagee
        VideoMetadata result = metadata.get(videoId);

        nbReaders--;
        if (nbReaders == 0) notifyAll();
        return result;
    }

    // Mise a jour des metadonnees (doit etre exclusif)
    public synchronized void updateMetadata(String videoId,
                                              VideoMetadata newData)
            throws InterruptedException {
        nbWritersWaiting++;        // annonce l'intention d'écrire
        while (writing || nbReaders > 0)
            wait();
        nbWritersWaiting--;        // on va maintenant écrire
        writing = true;

        // Acces exclusif a la ressource partagee
        metadata.put(videoId, newData);

        writing = false;
        notifyAll();
    }
}
Modifications par rapport à II.2 : (1) ajout du champ nbWritersWaiting, (2) condition de blocage de readMetadata élargie à || nbWritersWaiting > 0, (3) incrément avant l'attente et décrément juste avant d'écrire dans updateMetadata.
II.4

Risques de famine & accès équitable

Priorité lecteurs → famine des rédacteurs. Si les lectures arrivent en continu, nbReaders ne revient jamais à 0, et un rédacteur attend indéfiniment.

Priorité rédacteurs → famine des lecteurs. Si les mises à jour arrivent en continu, nbWritersWaiting > 0 en permanence, et les lecteurs n'entrent jamais.

Solution équitable : maintenir une file FIFO globale. En Java, on peut utiliser ReentrantReadWriteLock(true) (mode fair) qui garantit l'ordre d'arrivée indépendamment du type d'opération, sans modifier la logique métier.
